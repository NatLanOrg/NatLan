Compiler for natural language

Input
Documentation

UML diagrams, requirements, product use cases. Markdown, UML XMI, drawio, MS Word.

Process
Compilation

Separate compilation per file, then linking across files. Surfaces ambiguity and contradictions.

Output
Build artifacts

A machine-readable graph of entities, requirements, relationships, and diagnostics.

Control
  • IDE via the Language Server Protocol
  • MCP to query the artifacts
  • CLI for compilation and artifact viewing
Usages
  • Code generation
  • Test generation
  • LLM static analysis

Pipeline

Compilation transforms the docs into an intermediate representation: a graph of requirements, the entities they describe, and the typed relationships between them. The compiler reads loose prose and does not constrain the English you write.

Downstream usages consume that graph. Code generation reads the requirements for an entity and writes its implementation. Test generation writes a test per requirement. LLM static analysis reasons over the whole graph to flag gaps before any code exists.

Changes to requirements

A requirement is the unit of change. When a requirement is edited, the artifacts that depend on it recompile, and the generated code and tests bound to it are regenerated. The graph keeps a stable identity for each entity, so downstream outputs track the requirement they came from across edits.

IDE language support

A language server surfaces the compiler inside the editor: go to definition and find references across documents, hover for an entity's assembled definition, and diagnostics inline as you write. Contradictions across files show up where you read them.

auth.md:12  warning  "session expires after 30 minutes"
billing.md:device.md:5  contradicts  "sessions never expire"
